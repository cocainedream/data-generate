
1.paymentFormDetails.documentNumber
- Если договор заполен не верно выводится сообщение  "Номер заполнен неверно." - не очевидно о чем речь
- Значение c типом number 1, 123456 обрабатывается как "VALIDATED"
2. paymentFormDetails.documentDate
- Не более 10 календарных дней от текущей даты. Если сегодня 13 число, то 03 число проходит валидацию
3. paymentFormDetails.paymentKind
- Принимает тип значения number и проходит валидацию 4 или 5
4. paymentFormDetails.amount
- Принимает строковые значения такие как "123456.78"
- Принимает отрицательные значения такие как -0.11
- Если передать true, "2024-08-06", "12345abc", "1,01", " ", null в БД не попадает никакой записи
5. paymentFormDetails.name
- Принимает значение null
- Может быть пустым
- Принимает запрещенные символы такие как ☃☠
- Максимальное количество может превышать 160 символов, в тесте использовалось 161
6. paymentFormDetails.accountNumber
- Принимает следующие значения: "4082500040000000йцу7", "40825000400000+-\\у7", "000000000000000000000", "1", "4082500040000000006744444", true, null,
7. paymentFormDetails.bankName
- Может быть пустым
- Принимает невалидные символы Invalid©Symbol
- Принимает более 125 символов
- Принимает null
- Принимает true
8. paymentFormDetails.bankBic
- Принимает null
- Принимает значение вроде "1", "1234567890", "авава",  - которого нет в справочнике существующих БИК
- Принимает -141478855, "-%4"
- Принимает "2024-08-06"
- принимает true, false
9. paymentFormDetails.bankCorrAccountNumber
- Принимает следующие значения: "4082500040000000йцу7", "40825000400000+-\\у7", "000000000000000000000", "1", "4082500040000000006744444", true, null,
10. paymentFormDetails.operationTypeCode
- Принимает любые значения
- Если передать null в БД не попадает никакой записи
11. paymentFormDetails.paymentPurposeCode
- Принимает null
- Принимает значения с типом number
12. paymentFormDetails.paymentPriority
- Принимает текстовые значения '1', '2'
- Если передать null, "@#", "9.00" true в БД не попадает никакой записи
